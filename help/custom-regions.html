Using custom regions

NextWayThere allows you to use custom regions, by loading GTFS files from your device. In browsers supporting the FileSystem API (most modern browsers), you can select a directory which contains the required GTFS files. The feed must have already been extracted from the zip file. NextWayThere will then scan the directory for the required files to ensure that they all exist. It will only read files with recognised GTFS-like names in the selected directory. Files are not written to, and anything unrecognised is ignored.

If everything required is in place, the files will be loaded into the browser's storage, as with other GTFS feeds. You can delete the files from your computer once this has happened if you want to reduce data duplication.

Currently, NextWayThere does not support combining files which are stored locally (although this feature is planned to exist in the future). If you want to combine files, you may wish to consider using a merge tool such as this one.
